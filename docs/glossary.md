# Glossary

| Term | Meaning |
|---|---|
| HTML | HyperText Markup Language. Defines the structure and content of a web page. |
| CSS | Cascading Style Sheets. Controls the appearance and layout of a web page. |
| JavaScript | A programming language used to add behavior and interactivity to a page. |
| DOM | Document Object Model. The browser's structured representation of the page. |
| Metadata | Information about the page, often placed in the HTML `head`. |
| Accessibility | Making websites usable by as many people as possible, including people using assistive technology. |
| Responsive design | Designing pages so they adapt to different screen sizes. |
| Form | A structure for collecting user input. |
| Label | Text associated with a form input so users understand what the input is for. |
| Alt text | Text description of an image for users who cannot see the image or when the image fails to load. |
| Viewport | The visible area of a web page in the browser window or device screen. |
| Favicon | The small icon shown in a browser tab or bookmark. |
| Repository | A Git-tracked project folder, often hosted on GitHub. |
| README | The main documentation file for a repository. |
| Changelog | A file that summarizes changes by version. |
